# 原型录屏脚本（90 秒版）

这份脚本适合你自己补录一版真人操作视频，也适合面试时口头讲解。

---

## 1. 录屏目标

在 90 秒内让老师快速理解三件事：

1. MindOS 服务的是 **多 Agent 工作流**；
2. 它解决的是 **上下文共享 + 判断沉淀**；
3. 它不是单页 Demo，而是一个 **有完整系统边界的产品**。

---

## 2. 推荐镜头顺序

### 镜头 1（0s - 12s）产品总览

画面：`assets/screenshots/00-product-loop.png`

口播：

> 我独立完成的这个项目叫 MindOS。它是一个面向多 AI Agent 场景的本地优先知识操作系统。它的核心不是聊天，而是把用户在工作中的判断沉淀成可复用资产，让不同 Agent 共享同一套上下文。

### 镜头 2（12s - 28s）首页工作台

画面：`assets/screenshots/01-home.png`

口播：

> 这里是首页工作台。用户不是进入一个聊天框，而是进入自己的知识系统。项目文档、规则、空间结构、搜索入口都在这里统一组织。

### 镜头 3（28s - 48s）AI 对话页

画面：`assets/screenshots/02-chat.png`

口播：

> 在 AI 对话页里，Agent 不是孤立回答，而是结合知识库、当前文件和规则来执行任务。这样用户之前写过的背景、规范和方法论都能直接复用，不用每次重新讲。

### 镜头 4（48s - 65s）Agent 管理 / Dashboard

画面：`assets/screenshots/03-dashboard.png` 或 `assets/screenshots/05-agents-page.png`

口播：

> 我把多 Agent 作为核心场景来设计，所以产品里有专门的 Agent 管理和可见性界面。用户可以统一查看不同 Agent 的接入状态、能力和配置，而不是只面对一个黑箱。

### 镜头 5（65s - 82s）Echo 反思页

画面：`assets/screenshots/04-echo.png`

口播：

> 这部分是 Echo，用来承接反思和认知沉淀。我的目标不只是让 AI 更会做事，也让用户自己的判断和方法论能沉淀下来，形成长期复利。

### 镜头 6（82s - 90s）收束

画面：回到 `assets/screenshots/00-product-loop.png`

口播：

> 所以这个项目最能代表我的一点是：我不是只用 AI 写代码，而是独立完成了从产品定义、交互设计到工程实现和系统治理的完整闭环。

---

## 3. 如果你要录真人版，建议补两个动作

1. 演示一个文件如何被 Agent 读取并参与回答；
2. 演示一个规则文件如何影响不同 Agent 的行为。

这样更能突出“判断沉淀层”的差异化。

---

## 4. 录制建议

- 分辨率：优先 1440p 或 1080p；
- 时长：控制在 60-90 秒；
- 语速：比平时略慢，保证每句话都能被记住；
- 结构：问题 → 方案 → 界面 → 价值，不要一上来就讲技术栈；
- 开头第一句话一定要先说定位，不要先说“这是我做的一个工具”。
