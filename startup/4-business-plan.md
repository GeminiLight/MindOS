## 一、执行摘要

### 公司概述

AI 工具正在裂变——写代码的、做设计的、搜信息的、管项目的，每个场景都有专门的 Agent。用户不是选择用多个 AI，是不得不用多个。但每个 AI 都是独立的孤岛：切工具重新讲背景，纠正了明天还犯，偏好设了换一个又要重设。

MindOS 是你和所有 AI 之间的**判断沉淀层**。你写下判断，所有 AI 遵守。你纠正 AI，纠正变成规则。飞轮转起来后，AI 越用越准，人的思维也越来越清晰。

**品类：** 认知外化平台。不是笔记工具，不是 AI 记忆，不是 Agent 编排。MindOS 告诉 Agent 你是谁。

### 产品状态

- **阶段：** P1.5 进行中
- **版本：** v0.6.81
- **已实现：** GUI 工作台 + MCP Server + 一键导入 + ACP/A2A 双协议 + Workflow 编排 + Skills 系统 + Echo 回响 + 桌面端（三平台）+ CLI + 新手引导
- **发布：** npm 已发布，GitHub 开源
- **用户：** 内测群约 30 人
- **域名：** mindos.you

### 飞轮

```
1 痛点入口    "别重复交代"           -> 安装 MindOS
2 判断积累    纠正沉淀为规则          -> 规则越来越多
3 AI 变准     Agent 复用规则          -> 用户更依赖

```

## 二、问题与市场机会

### 三层问题

1. **使用割裂** — 每个 AI 是孤岛，你在孤岛之间来回搬运上下文
2. **经验在蒸发** — 对话中的判断关掉就没了，没有系统在收集
3. **成长不复合** — 你在变强，但进步散落在 100 个对话里，从未复合

### 根因

关系搞反了。人在伺候 AI（手动维护 .cursorrules / CLAUDE.md），而不是 AI 适配人。

### 市场时机

- MCP 2025 年标准化，多 Agent 工作流 2025-2026 成为常态
- "跨 Agent 判断沉淀"品类目前空白
- 开发者是入口（280 万多 Agent 用户），知识工作者是终局（5 亿）

> 注："多 Agent 判断沉淀"是新品类，尚无第三方报告。以上为 bottom-up 假设。

## 三、产品与解决方案

### 解决方案

- **判断沉淀** — 纠正和偏好自动变成规则
- **跨 Agent 复用** — MCP + ACP 双协议，所有 Agent 读同一套规则
- **人可治理** — 纯 Markdown，你随时审查、修正、删除
- **5 分钟 Aha** — 安装后导入已有规则文件，Agent 立刻遵守

### 核心概念三层

- **Space**（结构层）— 按维度组织知识
- **Instruction**（控制层）— 所有 Agent 遵守的全局规则
- **Skill**（执行层）— 判断沉淀为可复用的执行手册

## 四、商业模式

### 商业化逻辑

MindOS 的 Agent 主动性分四层。层次 1-3（管理知识、直接交付、预测意图）的基础能力已开源——这是获客杠杆，不是让利。层次 4（反哺认知）是闭源的付费核心——这是用户在任何其他工具里得不到的独占价值。

```
开源（连接层）         → 驱动采纳         → 用户越多飞轮越快
闭源（认知层）         → 驱动付费         → Pro 收入
数据+算法壁垒          → 驱动护城河       → 用越久越离不开
```

**Free 给"做事的能力"。Pro 给"认识自己的能力"。Team 给"认识团队的能力"。**

### 三层 SaaS 订阅（按 Agent 主动性四层对齐）

```
Free（开源）                     Pro $8/月                       Team $16/人/月
─────────────────────────────────────────────────────────────────────────────────

层次 1: 管理知识
  ✅ MCP Server（24 工具）
  ✅ GUI + 11 渲染器
  ✅ 一键导入 + AI Organize
  ✅ Space / Instruction / Skills
  ✅ 1 Agent 连接               → 不限 Agent 连接

层次 2: 直接交付
  ✅ Bootstrap                   → 经验自动编译                  → 团队 Skill 共享
  ✅ Skills（手写）              → Skills（AI 自动生成）         → 方法论市场

层次 3: 预测意图
  (基础功能)                     → 上下文预加载                  → 团队级上下文路由
                                 → 规则缺口提示

层次 4: 反哺认知 ⭐
  (无)                           → 纠正热力图                   → 团队认知报告
                                 → 认知时间线                    → 跨成员盲区分析
                                 → 盲区地图                      → 方法论对齐度
                                 → 规则矛盾检测
                                 → 知识库健康度
                                 → 成长报告（月度）              → 团队成长看板
                                 → 碎片→原则提炼
```

### 付费触发点

| 触发点 | 时机 | 转化逻辑 |
|--------|------|---------|
| **第 2 个 Agent** | 用户连接第 2 个 Agent | Free 只支持 1 个，多 Agent 是核心痛点 |
| **第 20 条规则** | 规则积累到一定量 | 弹出"你的规则可以归纳为 3 条原则"预览，详细版需 Pro |
| **第 1 个月** | 使用满 30 天 | 生成"认知成长报告"预览，完整版需 Pro |
| **规则矛盾** | 检测到矛盾时 | 提示"发现 2 条规则矛盾"，查看详情需 Pro |

### Open Core 策略

| 层 | 开源/闭源 | 内容 | 理由 |
|----|----------|------|------|
| 连接层 | **永远开源** | MCP Server、CLI、基础 GUI、Space/Instruction | 驱动采纳，19 个 Agent 都能连 |
| 执行层 | **部分开源** | 手写 Skills 开源，AI 自动生成 Skills 闭源 | 基础能力免费，智能增值收费 |
| 认知层 | **闭源** | 反哺认知全部功能（热力图、时间线、盲区、成长报告等） | 独占价值，竞品无法替代 |

| 指标 | 目标 |
|------|------|
| Free → Pro 转化率 | 3-5% |
| Pro 月留存 | 85% |
| LTV/CAC | > 3x |

## 八、风险与应对

### 风险一：PMF 验证

> P0.5 验证冲刺已于 2026-04 完成。详细结果待补充到本章节。

### 风险二：平台风险

MCP 是 Anthropic 发布的协议。对冲：MindOS 做的是"人的记忆"（透明），大厂做的是"AI 的记忆"（黑箱），方向相反。

### 风险三：一人公司天花板

种子轮 40% 用于招人。在融到钱之前，所有事情只能一个人扛。

### 风险四：Scope 过大

功能面太宽，需要聚焦。每个功能做到了"能用"但没做到"极致"。

## 九、愿景

MindOS 的终局不是一个工具——是每个人都有一个随身的、不断进化的认知伙伴。你的判断不再随着对话消失，而是持续积累、被所有 AI 复用、反哺你自己的成长。

人机共演化：AI 越来越懂你，你越来越懂你自己。
