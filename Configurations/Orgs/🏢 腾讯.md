# 🏢 腾讯

## 1️⃣ 安装内部 Agent 工具

```bash
npm install -g --registry=https://mirrors.tencent.com/npm @tencent/claude-code-internal
npm install -g --registry=https://mirrors.tencent.com/npm @tencent/gemini-cli-internal
npm install -g --registry=https://mirrors.tencent.com/npm @tencent/codex-internal
```

## 2️⃣ 配置 Alias

```bash
RC_FILE=$([ "$SHELL" = "/bin/zsh" ] && echo ~/.zshrc || echo ~/.bashrc)
cat << 'EOF' >> "$RC_FILE"
alias cci='claude-code-internal'
alias gci='gemini-cli-internal'
alias cxi='codex-internal'
EOF
source "$RC_FILE"
```

```bash
mkdir -p ~/.claude-internal/skills && cp -r ~/.agents/skills/* ~/.claude-internal/skills/

mkdir -p ~/.codex-internal/skills && cp -r ~/.agents/skills/* ~/.codex-internal/skills/

mkdir -p ~/.gemini-internal/skills && cp -r ~/.agents/skills/* ~/.gemini-internal/skills/
```


## 3️⃣ 内部工具配置文件路径

腾讯内部 Agent 工具的配置目录与标准工具不同：

| 工具 | 配置文件路径 |
|------|------------|
| claude-code-internal | `~/.claude-internal/` |
| gemini-cli-internal | `~/.gemini-internal/` |
| codex-internal | `~/.codex-internal/` |

全局协议（CLAUDE.md 等）、settings、skills 均写入对应的内部路径，而非 `~/.claude/` 等标准路径。

因此，应在 `Agents/🤖 Agent 全局协议.md` 的 Agent 文件路径表格中，追加腾讯内部工具对应的路径说明。
