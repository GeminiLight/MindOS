## 渠道策略（按优先级排序）

### 第一梯队：开发者社区（当前重点）

| 渠道 | 策略 | 指标 |
|------|------|------|
| **GitHub** | README 优化 + 持续 commit + 回复 issues | Stars, Forks, Issues 数 |
| **npm** | 包名可搜索、关键词优化 | 周下载量 |
| **Hacker News** | Show HN 帖子（时机：有明确 demo 时） | 首页停留时间、评论数 |
| **X / Twitter** | 工作流 demo 短视频 + Agent 相关话题参与 | Impressions, Profile visits |

### 第二梯队：中文社区

| 渠道 | 策略 | 指标 |
|------|------|------|
| **即刻** | AI 工具 / 效率工具圈子，发使用场景 | 点赞 + 评论 |
| **小红书** | "AI 工作流"类图文，视觉化展示 | 收藏 + 关注 |
| **V2EX** | 技术向介绍帖 | 回复数 |
| **知乎** | "AI Agent 记忆管理"相关问题回答 | 赞同 + 关注 |

### 第三梯队：被动获客

| 渠道 | 策略 |
|------|------|
| **Agent 厂商集成市场** | MCP Server 注册到 Claude / Cursor 等生态 |
| **技术博客** | 写 MCP 协议、Agent 知识管理相关文章 |
| **开源项目交叉推广** | 与 Obsidian 插件、Agent 框架等社区互动 |

## 内容策略

### 核心内容类型

| 类型 | 示例 | 目标 |
|------|------|------|
| **痛点共鸣** | "你的 5 个 AI Agent 各记各的，互相不认识" | 引起兴趣 |
| **工作流 demo** | 录屏：Claude Code 通过 MCP 读写 MindOS 知识库 | 降低理解门槛 |
| **对比测评** | MindOS vs Obsidian + AI 插件 vs Notion AI | SEO + 搜索流量 |
| **用户故事** | 内测用户的真实使用场景 | 社会证明 |

### 内容生产节奏

- 每周 1 条社交媒体内容（X / 即刻 / 小红书轮换）
- 每 2 周 1 个工作流 demo 视频
- 每月 1 篇深度博客
